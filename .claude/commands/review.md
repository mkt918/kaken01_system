## チェック項目

### コンテンツ品質
- 説明が生徒のレベルに合っているか（高校生・初学者向け）
- 例題・ドリルの難易度が適切か
- 前の週との繋がりが自然か

### 技術的な確認
- Markdown の書式が正しいか
- HTMLファイルから正しく読み込めるか（パス・ファイル名）
- Tailwind CSS のクラスに誤りはないか

## 手順
1. `git diff HEAD` で変更内容を確認
2. 変更されたファイルを読み込む
3. 上記チェック項目で問題点・改善点を重要度順にリストアップ
4. 修正が必要な箇所は具体的に指摘する（修正は行わない）
