# W1: プログラミングとは ― アルゴリズムとフローチャートの世界

コンピュータは、指示した通りにしか動きません。
「どんな手順で、何をするか」を正確に組み立てることが**プログラミング**です。
今週は、プログラミングの根本にある「**アルゴリズム**」と「**フローチャート**」を徹底マスターします。

---

## レッスン 1: プログラミングとは？

### プログラムとは「手順書」

プログラムとは、コンピュータへの**命令の手順書**です。

```
【料理レシピ（プログラム）の例】
1. お鍋に水を入れる
2. 火をつける
3. 沸騰したら麺を入れる
4. 3分待つ
5. 具材を入れる
6. 完成！
```

順番を間違えると失敗します。コンピュータへの命令も同じです。
「水を入れる前に火をつける」ようなミス＝**バグ（Bug）**といいます。

### プログラミング言語とは？

コンピュータは日本語や英語を直接理解できません。
そこで、コンピュータが理解できる「**プログラミング言語**」で書きます。

| 言語 | 用途 | 特徴 |
|------|------|------|
| **Python** | AI・データ分析・Web | 初心者に最も優しい |
| JavaScript | Webサイト（動き） | ブラウザで動く |
| Java | 業務システム・Android | 大規模開発に強い |
| C言語 | OS・組み込み機器 | 高速・低レベル制御 |

この授業では **Python** を使います。

## レッスン 2: アルゴリズムとは？

**アルゴリズム（Algorithm）**とは、「問題を解くための手順」のことです。

### 日常のアルゴリズム例

**「ジュースを自販機で買う」アルゴリズム：**

```
1. お金を入れる
2. ほしいジュースのボタンを押す
3. ジュースが出てくる
4. おつりを受け取る
```

この4ステップがアルゴリズムです。順番を変えると失敗します。

### アルゴリズムの3つの条件

| 条件 | 説明 | 例 |
|------|------|----|
| **有限性** | いつか必ず終わる | 無限ループはNG |
| **明確性** | 曖昧さがない | 「少し待つ」ではなく「3秒待つ」 |
| **入出力** | 入力を受けて結果を出す | 数字を入れたら答えが出る |

## レッスン 3: フローチャートの基本記号

フローチャートはアルゴリズムを**図で表したもの**です。
世界中のプログラマーが使う「共通言語」です。

### 基本4記号

```
┌──────────┐
│   楕 円   │  ← 開始・終了（スタートとゴール）
└──────────┘

┌──────────┐
│   四 角   │  ← 処理（何かをする）
└──────────┘

   /◇◇◇◇\
  < 菱  形  >  ← 判定（Yes/No で分岐）
   \◇◇◇◇/

      ↓        ← 矢印（処理の流れ）
```

### 記号の使い方まとめ

| 記号 | 形 | 使う場面 | 例 |
|------|----|----------|----|
| 端子 | 楕円 | 開始・終了 | `START` / `END` |
| 処理 | 四角 | 計算・代入・表示 | `x = 5 + 3` |
| 判断 | 菱形 | 条件分岐 | `x > 10?` |
| 矢印 | → ↓ | 処理の順序 | 上から下へ |

## レッスン 4: プログラムの3つの基本構造

どんなプログラムも、この3つだけで作られています。

### 構造① 順次（じゅんじ）― 上から順番に

最もシンプルな構造。命令を上から下へ順番に実行します。

**フローチャート:**
```
   [START]
      ↓
[x = 10 を代入]
      ↓
[y = 20 を代入]
      ↓
[z = x + y を計算]
      ↓
[z を表示する]
      ↓
    [END]
```

**Python コード:**
```python
x = 10
y = 20
z = x + y
print(z)   # 30
```

---

### 構造② 選択（せんたく）― 条件で分かれる

「もし〜なら〜する、そうでなければ〜する」という分岐です。

**問題：「気温が25度以上なら『暑い』、未満なら『涼しい』と表示する」**

**フローチャート:**
```
      [START]
         ↓
  [気温 temp を入力]
         ↓
    /temp >= 25\
   < YES     NO >
    \        /
     ↓      ↓
  [「暑い」] [「涼しい」]
  を表示    を表示
     \      /
      ↓    ↓
       [END]
```

**Python コード:**
```python
temp = int(input("気温を入力："))

if temp >= 25:
    print("暑い！")
else:
    print("涼しい！")
```

**実行例:**
```
気温を入力：30
暑い！
```

---

### 構造③ 反復（はんぷく）― 繰り返す

同じ処理を条件が満たされるまで繰り返します。

**問題：「1から5まで数字を順番に表示する」**

**フローチャート:**
```
    [START]
       ↓
   [i = 1]
       ↓
  /i <= 5\   NO → [END]
  \ YES  /
     ↓
  [i を表示]
     ↓
  [i = i + 1]
     ↓
（判定に戻る）
```

**Python コード:**
```python
i = 1
while i <= 5:
    print(i)
    i = i + 1
```

**実行結果:**
```
1
2
3
4
5
```

## ドリル 1-2：フローチャートを読む（ループ版）

```
    [START]
       ↓
   [n = 0]
   [s = 0]
       ↓
  /n < 4\   NO → [s を表示] → [END]
  \ YES /
     ↓
  [s = s + n]
     ↓
  [n = n + 1]
     ↓
（判定に戻る）
```

最後に `s` の値はいくつになりますか？

<details>
<summary>答えを見る</summary>

```
n=0: s = 0+0 = 0、n→1
n=1: s = 0+1 = 1、n→2
n=2: s = 1+2 = 3、n→3
n=3: s = 3+3 = 6、n→4
n=4: 4 < 4 は偽 → ループ終了
```

**答え：s = 6**（0+1+2+3 = 6）

</details>

## ドリル 1-5：バグを探せ！

以下のフローチャートには **論理的なバグ** が1つあります。見つけてください。

**「1から10の合計を求める」つもりのフローチャート:**

```
    [START]
       ↓
   [i = 1]
   [sum = 0]
       ↓
  /i < 10\   NO → [sum を表示] → [END]
  \ YES  /
     ↓
  [sum = sum + i]
     ↓
  [i = i + 1]
     ↓
（判定に戻る）
```

<details>
<summary>答えを見る</summary>

**バグ：** 判定が `i < 10` になっている（`i = 10` のとき処理されない！）

**正しくは `i <= 10`** にする必要があります。

`i < 10` だと **1〜9 の合計（= 45）** しか求められず、10が含まれません。
`i <= 10` にすれば **1〜10 の合計（= 55）** が求められます。

</details>

## ミニプロジェクト：買い物の会計プログラム

「商品名・値段・個数を入力すると、合計金額と消費税込みの金額を表示する」
プログラムのフローチャートを描き、Pythonコードを書きましょう。

### 要件
- 商品名（文字列）・値段（整数）・個数（整数）を入力する
- 合計 = 値段 × 個数 を計算する
- 税込み = 合計 × 1.10 を計算する（端数切り捨て）
- 結果を見やすく表示する

### フローチャート

```
       [START]
          ↓
  [商品名・値段・個数を入力]
          ↓
  [合計 = 値段 × 個数]
          ↓
  [税込み = 合計 × 1.10（切り捨て）]
          ↓
  [商品名・合計・税込みを表示]
          ↓
        [END]
```

<details>
<summary>解答例を見る</summary>

```python
# 入力
name   = input("商品名を入力：")
price  = int(input("値段（円）を入力："))
count  = int(input("個数を入力："))

# 計算
total  = price * count
taxed  = int(total * 1.10)   # 端数切り捨て

# 表示
print("=" * 30)
print(f"  商品名 : {name}")
print(f"  値 段  : {price} 円")
print(f"  個 数  : {count} 個")
print("-" * 30)
print(f"  合 計  : {total} 円（税抜き）")
print(f"  税込み : {taxed} 円")
print("=" * 30)
```

**実行例:**
```
商品名を入力：ノート
値段（円）を入力：150
個数を入力：3
==============================
  商品名 : ノート
  値 段  : 150 円
  個 数  : 3 個
------------------------------
  合 計  : 450 円（税抜き）
  税込み : 495 円
==============================
```

</details>
