## 📝 小テストの解答と解説（全5問）

**Q1.** パソコンの画面に文字を表示させるための命令はどれ？

- **正解:** **B. `print()`**
- **解説:** `show()` や `output()` はPythonに存在しない命令です。

**Q2.** 文字を表示させるとき、正しく囲まれているのはどれ？

- **正解:** **C. `print('こんにちは')`**
- **解説:** シングルまたはダブルクォーテーションで囲むのが正解。囲まないと変数として扱われ `NameError` になります。

**Q3.** プログラミングにおける `=`（イコール1つ）の正しい意味はどれ？

- **正解:** **B. 右のものを、左の箱に入れる（代入する）という意味**
- **解説:** 数学の「等しい」と混同させないことが大切です。比較には `==` を使います（来週以降）。

**Q4.** 変数 `item` の中身を表示させる正しい書き方はどれ？

- **正解:** **B. `print(item)`**
- **解説:** `print("item")` だと、ただの「i・t・e・m」という4文字が表示されます。クォートの有無で意味がまったく変わることを強調してください。

**Q5.** 変数の中身を文章に埋め込む「f文字列」の正しい書き方はどれ？

- **正解:** **A. `print(f"これは {item} です")`**
- **解説:** `f` の位置はクォーテーションの外（左側）、変数を埋め込む場所は波カッコ `{ }` を使うのが大鉄則です。

## 📊 発展指導案：アルゴリズムとフローチャート（オプション・今後のステップへの架け橋）

- 次週以降の「条件分岐（if文）」「繰り返し（ループ）」への自然な架け橋となる

### 💡 指導時のコツ

#### 1. 「自分の生活の中でのアルゴリズム」を先に体験させる

黒板に以下の問題を書き、生徒に一緒に「フローチャート風」に考えさせます：

```
「朝、起床から学校を出発するまでのアルゴリズムを書いてみよう」
├─ 目覚まし時計が鳴った
├─ ベッドから起きる
├─ 朝食を食べるか？
│  ├─ YES → 食べる → 時間チェック
│  └─ NO → スキップ
├─ 時間に余裕があるか？
│  ├─ YES → ゆっくり準備
│  └─ NO → 急いで準備
└─ 家を出る
```

この体験を通じて、「アルゴリズム＝生活と同じ『流れ』」という気づきが生まれます。

#### 2. フローチャート記号を「絵」として理解させる

- 楕円 🟢 ＝「開始・終了」＝ **人生のスタートとゴール**
- 四角 □ ＝「処理」＝ **何かをやる**
- 菱形 ◇ ＝「判定」＝ **Yes/No で分かれ道**

黒板に大きく描き、生徒のノートにも描かせてください。

#### 3. 「フローチャート → コード」の変換を見せる

フローチャートをコードに直すとき、**逐一対応**を意識させます：

```
フローチャート上の             Pythonコード上の
【四角：処理】        →        【変数代入・print()】
   ↓
【菱形：判定】        →        【if 条件：】
   /\
 /  \
```

#### 4. バグ（プログラムの間違い）を「アルゴリズムレベル」で考えさせる

「コードが動かない」という相談を受けたとき：
- コード文法のエラー（`SyntaxError` など）だけでなく
- **「アルゴリズム自体が間違っている」可能性**を気づかせる

例えば、FizzBuzz で「15の倍数」の判定が後ろにあるコードは間違います：

```python
# 間違ったアルゴリズム
if i % 3 == 0:
    print("Fizz")      # 9, 12, 15... すべてここで終わり
elif i % 15 == 0:
    print("FizzBuzz")  # ここには絶対到達しない！
```

こうした「**論理的な誤り**」を、フローチャートを描かせることで防げます。

### 📌 注意点と対応策

#### ❌ よくある失敗

1. **フローチャートが複雑すぎる**
   - 最初は「順序」「1つの判定」「簡単なループ」のみにしてください
   - 複数の判定が絡む FizzBuzz は、基本を習得してから

2. **生徒がフローチャートを描くのを嫌がる**
   - 「面倒だ」という声が出ます
   - **「フローチャートを描くと、バグが減って、プログラムが速く書ける」** と実感させることが鍵
   - ペアで一人が フローチャート、一人がコード というやり方も効果的

3. **判定の矢印が複雑になる**
   - 2つ以上の判定がある場合、矢印が絡み合います
   - 「YES の矢印は右に、NO の矢印は下に」などルール化してください

#### ✅ 対応策

- 大事なのは「**正確さ**」より「**論理の流れが見える**」ことです
- 最初の生徒のフローチャートに「少し余計な矢印があってもいい」と許容してください
- むしろ「自分のアルゴリズムを図で説明できた」という体験が重要です
