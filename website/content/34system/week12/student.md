Week 9でデータを「追加（INSERT）」し、Week 10〜11で「見る（SELECT）」ことを学びました。

## 🕒 レッスン 1: データを書き換える `UPDATE`

勇者がレベルアップしてHPが増えた時や、引越しで住所が変わった時に使う魔法が **`UPDATE`（アップデート）** です。

### 💻 ハンズオン（UPDATEで書き換える）

先生が用意した事前準備コードをRunして、勇者テーブル（`heroes`）を作成してください。

```sql
/* UPDATE テーブル名 SET 書き換えたい列 = 新しい中身 WHERE 誰を？; */
UPDATE heroes
SET hp = 999
WHERE name = 'アーサー';

/* 本当に書き換わったか確認！ */
SELECT * FROM heroes;
```

> **💡 解説: `UPDATE` のルール**
>
> 1. `UPDATE` の後にテーブル名を指定します。
> 2. `SET` の後に `列名 = 新しいデータ` を書きます。
> 3. **絶対に `WHERE` で「誰を書き換えるか」を指定**してください！（理由はレッスン3で体験します）

### ✍️ ドリル 1-1：レベルアップ

`heroes` テーブルの中にいる「ランスロット」の `level` を `50` に書き換えてください。

```sql
UPDATE heroes
SET level = 50
WHERE name = 'ランスロット';

SELECT * FROM heroes;
```

> **確認：** ランスロットのlevelだけが50になり、他の勇者のlevelは変わっていませんか？

## 🕒 レッスン 2: データを消し去る `DELETE`

モンスターを倒した時や、ゲームを退会したユーザーのデータを完全に削除するのが **`DELETE`（デリート）** です。

### 💻 ハンズオン（DELETEで削除する）

```sql
/* DELETE FROM テーブル名 WHERE 誰を？; */
DELETE FROM heroes
WHERE name = '最弱の村人';

SELECT * FROM heroes;
```

> **確認：** 「最弱の村人」のレコードだけが消えて、他の4人は残っていますか？

---

## 🕒 レッスン 3: 恐怖の大事件！「WHERE忘れの全滅」を体験せよ

プロのエンジニアが一番恐れている事件、それが**「UPDATEやDELETEをする時に、間違えて WHERE を書き忘れること」**です。

### 💻 恐怖のハンズオン（最悪のバグを起こす）

もし、`WHERE`（誰を）を書き忘れるとどうなるのでしょうか…？以下のSQLを実行して、すぐに `SELECT * FROM heroes;` を見てください。

```sql
/* 【危険】WHERE を書かずに UPDATE してしまう！ */
UPDATE heroes
SET hp = 1;

SELECT * FROM heroes;
```

> **⚠️ 結果の確認**
> 全員のHPが `1` になってしまいました！！
> これが実際の銀行のデータベースで起きたら… 全員の預金が1円になります。絶対にやってはいけない操作です。

**✨ 解決法：** 事前準備コードをもう一度 Run（実行）して、壊れたデータを真っさらな初期状態にリセット（復活）させましょう！

> **DELETE でも同じことが起きます！**
> `DELETE FROM heroes;` のように `WHERE` なしで実行すると、テーブルの全データが一瞬で消えます。
> `UPDATE` も `DELETE` も、必ず `WHERE` とセットで使う習慣をつけましょう。

## 🚀 今日の総仕上げ：ミニプロジェクト「RPGステータス全回復ポイント」

ダンジョンの奥にある「回復の泉」を踏みました。
`heroes`（勇者たち）パーティのうち、レベルが10以下のやられそうなキャラのHPだけを特別に「500」に大回復してあげましょう。

### 📝 ミッション

以下のステップで順番に作ってみましょう。

**ステップ1：** まず `SELECT * FROM heroes;` で現在の全データを確認してください。

**ステップ2：** `UPDATE` を使って `heroes` テーブルを更新します。`SET` で `hp` を `500` にします。

```sql
UPDATE heroes
SET hp = 500
WHERE level <= 10;
```

**ステップ3：** 再び `SELECT * FROM heroes;` で確認して、レベル10以下のキャラのHPだけが500になっているか確認しましょう。

#### 完成イメージ

```text

1  | アーサー     | 15    | 200   ← 変わっていない（level 15）
2  | ランスロット  | 50    | 500   ← 500に回復！（level <= 10 だったため）
3  | ガラハッド   | 5     | 500   ← 500に回復！（level 5）
4  | マーリン     | 12    | 100   ← 変わっていない（level 12）
```

## 🚀 早く終わった人へ：ミニプロジェクト「大掃除アプリ（ゴーストユーザーの一掃）」

あなたはオンラインゲームの管理者です。サーバーが重くなってきたため、もう何年もログインしていない幽霊ユーザー（ゴーストユーザー）のデータを一斉に削除して大掃除をします。

事前準備の `users`（ユーザー）テーブルを使います。
カラムは `id`, `name`, `last_login_days`（最後にログインしてからの経過日数）です。

### 📝 ミッション（ゴーストユーザー一掃）

**ステップ1：** まず `SELECT * FROM users;` で現在の全ユーザーを確認します。

**ステップ2：** 「最後にログインしてからの日数（`last_login_days`）」が **365以上（1年以上放置）** になっているユーザーを `DELETE` で削除します。

```sql
DELETE FROM users
WHERE last_login_days >= 365;
```

**ステップ3：** 再び `SELECT * FROM users;` で確認して、放置ユーザーが綺麗サッパリ消えて、最近遊んでいるユーザーだけが残っていることを確認してください！

**ステップ4：** 削除されたユーザーと残ったユーザーを元データと見比べて、正しく消えているか確認しましょう。
