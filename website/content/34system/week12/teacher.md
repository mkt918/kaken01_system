# 指導案: Week 12 「データを操る神の手（UPDATE・DELETE）」

## 🎯 今週のねらいと授業のゴール

データの「更新（UPDATE）」と「削除（DELETE）」を習得し、SQLの基本4大命令である「CRUD（Create, Read, Update, Delete）」を制覇します。
最大のハイライトは「WHERE句を忘れた時の全体上書き・全削除の恐怖」を味わわせることです。実社会のシステム障害の多くがこれで起きている危機感を伝えましょう。

```sql
CREATE TABLE heroes (id INTEGER, name TEXT, level INTEGER, hp INTEGER);

```

## ⏱️ タイムテーブル (180分)

### 1️⃣ 更新の魔法（UPDATEとSET） (0分〜40分)

- **【解説】**: 「スマホのマイページでプロフィール画像を変更したとき、裏側ではこのUPDATEが走っているんだよ」と身近な例で説明します。
- **【演習】**: 事前準備を実行後、「レッスン1」と「ドリル1-1」をやらせます。
- **【指導のポイント】**:
  - `UPDATE` → `SET` → `WHERE` の順番が覚えにくいので、「**ア**ップデート！ **セ**ット！ **ウェ**ア！」とリズムで頭に入れさせます。

### 2️⃣ 削除の魔法（DELETE） (40分〜70分)

- **【解説】**: アカウント退会処理などの裏側です。「DROP TABLE（テーブルごと消す）とは違い、枠組みは残して中身の行だけを消すんだよ」と補足できると素晴らしいです。
- **【演習】**: 「レッスン2」を実施します。
- **【注意点】**: `DELETE` には `SET` が不要なので `DELETE FROM heroes WHERE ...` とシンプルに書けることを強調します。

### 3️⃣ 恐怖のWHERE忘れ体験 (70分〜100分) ※その後休憩10分

- **【解説】**: この授業のメインイベントです。「IT業界の最も恐ろしい怪談をしよう...」と前置きして始めます。
- **【演習】**: 「レッスン3」を全員一斉に実行させます。
- **【大パニックのコントロール】**:
  - 全員のHPが1になった瞬間、「ああああ！終わった！」というリアクションを引き出します。銀行の金利計算ミスや、ソシャゲの配布ミスなどは大体これが原因であることを語り、プログラマーの責任の重さを伝えます。
  - その後、「大丈夫、バックアップがある」と言って、事前準備コードを再Runさせ、復旧（ロールバック）の体験もさせます。

### 4️⃣ 総仕上げ：ミニプロジェクト (110分〜170分)

- **【総合演習】**: 「RPGステータス全回復ポイント」
  - ミッションを3ステップに分解しているので、1ステップずつ確認しながら進めさせてください。
  - ピンポイントではなく、`WHERE level <= 10` という「広範囲の条件」に対するUPDATEです。一気に複数人のデータがガシャン！と書き換わる快感を味わせてください。
- **【早く終わった人向け（ゴーストユーザー一掃）】**:
  - `WHERE last_login_days >= 365` を使った広範囲削除です。ミッションを4ステップで分解しています。
  - 不要なデータがすべて消え去ることで、データベースの「管理」をしている実感が湧きます。管理者の特権操作の醍醐味です。

## 📝 小テストの解答と解説（全5問）

**Q1.** すでにあるデータを「新しい内容に書き換える（更新する）」ための命令文はどれ？

- **正解:** **B. `UPDATE`**

**Q2.** テーブルからデータを完全に「削除して消し去る」ための命令文はどれ？

- **正解:** **A. `DELETE FROM`**

**Q3.** `UPDATE` でデータを書き換える時、「新しい内容はこれだよ」と指定するための中間のキーワードは？

- **正解:** **B. `SET`**（「セットする」という意味です）

**Q4.** データベースの世界で最もやってはいけない致命的なミス（全滅バグ）の原因は何？

- **正解:** **C. `UPDATE` や `DELETE` で `WHERE` を書き忘れる**
- **解説:** これをやらかすと、最悪の場合ニュースに載ります。それくらい強い権限を持っている証拠だと説明してください。

**Q5.** 次のSQLを実行するとどうなりますか？ `DELETE FROM monsters;`

- **正解:** **B. `monsters` テーブルの中のデータがすべて消える**
- **解説:** WHEREがないので、「モンスター全部」を対象に削除が実行されてしまいます。
