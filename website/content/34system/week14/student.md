# Week 14: 別々のデータをくっつける「JOIN」とStep2総まとめ

データベースの世界には「同じ情報を何度も書かない」という美しいルールがあります。
そのため、ゲームのデータも「勇者テーブル」「武器テーブル」のように別々の箱に分けられています。
今週は、別々に分けられた箱を**「1つに合体（結合）させる」最強の魔術（JOIN）** を学び、1.5ヶ月に及ぶ「SQL基礎（Step2）」を総決算します！

## 🕒 レッスン 1: テーブルが分かれている理由

もし1つのテーブルに全部書いたらどうなるでしょう？
`名前: アーサー, 武器: 鉄の剣, 武器の攻撃力: 10, キャラHP: 100`  
これだと、「鉄の剣」の攻撃力が15に変わった時、鉄の剣を持っている100人全員のデータをアップデートしなければならず、バグの温床になります。
だからプロは**「キャラ」と「武器」のテーブルを別々に作る**のです。

## 🕒 レッスン 2: 2つのテーブルを結合する `JOIN`（ジョイン）

別々のテーブルを「共通のID」を鍵にしてガッチャンコ！とくっつける魔法が **`JOIN`（ジョイン）** です。

### 💻 ハンズオン（やってみよう）
先生が用意した事前準備コードをRunして、顧客テーブル（`customers`）と、注文履歴テーブル（`orders`）を作成してください。

```sql
/* 注文履歴（orders）に、顧客の名前（customers）をくっつける！ */
SELECT 
    orders.id,           /* 注文のID */
    customers.name,      /* お客さんの名前（くっつけた！） */
    orders.item_name     /* 買ったモノ */
FROM orders
JOIN customers 
  ON orders.customer_id = customers.id; /* くっつけるためのカギ（共通ID） */
```

> **💡 解説: `JOIN ... ON ...`**
> 1. `FROM orders`: 基本となる左側のテーブルを呼び出します。
> 2. `JOIN customers`: くっつけたい右側のテーブルを呼び出します。
> 3. `ON orders.customer_id = customers.id`: 「注文の顧客ID」と「顧客テーブルのID」が**一致する人同士を合体させろ！**という一番大切な条件です。

### ✍️ ドリル 2-1：キャラと武器を合体させろ！
勇者テーブル（`heroes`）と武器テーブル（`weapons`）があります。これをくっつけて、「誰が」「どの武器」を装備しているか出してください。

```sql
SELECT
    heroes.name,
    weapons.weapon_name
FROM heroes
________ weapons               /* weaponsテーブルをくっつける */
  ____ heroes.weapon_id = weapons.id;  /* 共通のIDで合わせる */
```

## 🚀 今日の総仕上げ1：ミニプロジェクト「お店屋さんの売上分析（JOIN）」

あなたはオンラインショップの店長です。「誰が一番たくさんお金を使ってくれたか（VIP客）」を分析したいのですが、名前は `customers` テーブルにしかなく、買った金額は `orders` テーブルにしかありません。

### 📝 ミッション
1. `orders` テーブルと `customers` テーブルを `JOIN` で結合してください。
2. そのうえで、お客さんの名前（`customers.name`）ごとに **`GROUP BY`** します。
3. そして、そのお客さんが買った金額の合計（`SUM(orders.price)`）を計算します。
4. 最後に、合計金額の **「大きい順（DESC）」** に並べ替えてください。
※過去の知識（`SUM`, `GROUP BY`, `ORDER BY`）と今日の `JOIN` がすべて繋がる超強力なSQLです！
