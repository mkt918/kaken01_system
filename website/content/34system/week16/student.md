## 🕒 レッスン 1: Pythonの世界にデータを呼び出す `SELECT` と `fetchall()`

データベースから探してきたデータは、そのままではPythonで使えません。
`fetchall()`（フェッチ・オール＝全部取ってくる）という命令を使って、Pythonが得意な「リスト（配列）」の形に変換して受け取ります。

### 💻 ハンズオン（やってみよう）
先生が用意した `shop.db` ファイルと同じフォルダで、以下のプログラムを実行してください。

```python
import sqlite3
connection = sqlite3.connect("shop.db")
cursor = connection.cursor()

# 1. 呪文を準備して唱える（この時点ではまだ結果を見ていない）
cursor.execute("SELECT * FROM items;")

# 2. 結果をすべてPythonのリストとして受け取る！
rows = cursor.fetchall()

# 3. forループを使って1行ずつ綺麗に表示する
for row in rows:
    print(f"商品コード: {row[0]}, 商品名: {row[1]}, 価格: {row[2]}円")

connection.close()
```

> **💡 解説: `row[1]` って何？**
> `fetchall()` で受け取ったデータは、それぞれの行が `(1, 'りんご', 150)` のような「タプル」という箱になっています。
> `row[1]` は「左から2番目（0から数えるので）のデータ」、つまり「りんご」を取り出しています！

## 🕒 レッスン 2: 【警告】恐怖のハッキング「SQLインジェクション」体験

ユーザーがキーボードで入力した文字（`input()`）を、そのままDBに入れるシステムを作ってみましょう。
実はこれ、**世界で一番危険なプログラム**です。

### 💻 恐怖のハッキング・ハンズオン
以下の危険なプログラムを実行し、名前を聞かれたら絶対に **`' OR '1'='1`** という謎の記号を入力してください。

```python
# 【絶対真似してはいけない危険なプログラム】
import sqlite3
connection = sqlite3.connect("secret.db")
cursor = connection.cursor()

# inputでパスワードを聞く
user_pass = input("秘密のパスワードを入力してください: ")

# inputで入れた文字を、そのままSQLに埋め込んでしまう（超危険！）
sql = f"SELECT * FROM secrets WHERE password = '{user_pass}';"
cursor.execute(sql)
results = cursor.fetchall()

print("見つかった秘密のデータ:", results)
connection.close()
```

**【結果の確認】**
パスワードを知らないはずなのに、秘密のデータが「全部」見えてしまいました！！
これが、実際に多くの大企業から個人情報が盗み出された**「SQLインジェクション（悪意あるコードの注射）」**という手口です。

## 🕒 レッスン 3: 最強の盾！安全機能「プレースホルダ `?`」

さっきの攻撃を防ぐために、プロは絶対に `f文字列` でパスワードを直接埋め込みません。
代わりに **`?`（プレースホルダ＝場所取りマーク）** を使います。

### 💻 ハンズオン（安全な入れ方）
上の危険なコードの `sql = ...` と `execute` の部分を、次のように書き改めてから、もう一度ハッキングの記号を入力してみてください。

```python
# 1. SQL文の「値」が入る場所は、すべて `?` にしておく
sql = "SELECT * FROM secrets WHERE password = ?;"

# 2. executeの横に、カンマで区切って (user_pass,) のようにタプルで渡す
cursor.execute(sql, (user_pass,))
```

こうすると、Pythonが「あ、これは入力されたただの文字だな」と判断して無害化（サニタイズ）してくれるため、ハッキングを防ぐことができます！

`input()` で聞いた「英単語」と「意味」を、安全な `?` を使ってデータベースに保存していくアプリを作ります。

### 📝 ミッション

### 📝 ミッション
