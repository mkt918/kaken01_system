## 🛠️ 事前準備（※授業の最初に先生が配布するファイル）
以下のPythonスクリプトを生徒に配布するか、初めにローカル環境で実行させ、`shop.db` と `secret.db` を生成させてください。

```python
import sqlite3

# shop.dbの準備
conn1 = sqlite3.connect("shop.db")
c1 = conn1.cursor()
c1.execute("CREATE TABLE IF NOT EXISTS items (id INTEGER, name TEXT, price INTEGER)")
c1.execute("INSERT INTO items VALUES (1, 'りんご', 150), (2, 'バナナ', 100), (3, '高級メロン', 3000)")
conn1.commit()
conn1.close()

# secret.dbの準備（ハッキング体験用）
conn2 = sqlite3.connect("secret.db")
c2 = conn2.cursor()
c2.execute("CREATE TABLE IF NOT EXISTS secrets (id INTEGER, name TEXT, password TEXT, top_secret TEXT)")
c2.execute("INSERT INTO secrets VALUES (1, 'たかやま', 'pass123', 'カツラである'), (2, '校長', 'koucho999', '実はゲーム好き')")
conn2.commit()
conn2.close()
```

### 1️⃣ Pythonにデータを吸い出す（fetchall） (0分〜40分)
- **【解説】**: 「SQLでSELECTした結果はあくまでDBの世界の出来事。それをPythonの世界に持ってくるのが `fetchall()`（全部取ってこい！）だよ」と物理的な移動のイメージで伝えます。
- **【演習】**: 「レッスン1」を実行させます。
- **【指導のポイント】**: 
  - `row[1]` や `row[2]` が何を指しているか（0番目がid、1番目がname...）を必ず復習させてください。Step 1（Week 5〜8）でやった「リスト（配列）の0から数えるルール」がここで活きます。

### 2️⃣ 恐怖のサイバー攻撃体験 (40分〜80分) ※その後休憩10分
- **【解説】**: いよいよSQLインジェクション体験です。パスワード入力欄に `' OR '1'='1` を入力させます。
- **【パニックのコントロール】**:
  - 用意された「校長の秘密（実はゲーム好き）」などが丸見えになった瞬間、教室が沸きます。
  - すかさず黒板で理由を解説します。「本来は `password = '入力'` になるところが、`password = '' OR '1'='1'` という文にすり替わった」と図説し、「1=1は絶対に正しいから、パスワードが違っても全部強制的に突破されてしまうんだ！」とハッカーの手口を鮮やかに謎解きしてください。

### 3️⃣ 最強の盾（プレースホルダ） (90分〜120分)
- **【解説】**: 「こんな単純な手口で世界中の会社が破られた。それを完全に防ぐのが `?` だ！」と救世主のように紹介します。
- **【注意点】**: 
  - 書き方が `cursor.execute(sql, (user_pass,))` のようにカッコが多くなります。特に `(user_pass,)` の**最後のカンマ（`,`）** は、Pythonで「要素1つのタプル」を作るための絶対条件なので、これを忘れるとエラーになることを強めに指導してください。

## 📝 小テストの解答と解説（全5問）

**Q1.** データベースから検索したデータを、Pythonのプログラムに「リスト」の形で受け取る時に使う命令はどれ？
- **正解:** **B. `fetchall()`** （Fetchは「取ってくる、連れてくる」という意味です）

**Q2.** `fetchall()` で受け取ったデータの1行分（`row`）は、どのような形に包まれていますか？
- **正解:** **A. タプル（例: `(1, 'りんご', 150)`）** （後から書き換えられない安全なリストの兄弟です）

**Q3.** ユーザーが画面から入力した文字に「悪意のある強力なSQLの断片」を紛れ込ませて、DBを破壊・盗聴するサイバー攻撃を何と呼ぶ？
- **正解:** **B. SQLインジェクション** （インジェクション＝注射。悪意のあるコードを注射されるイメージです）

**Q4.** SQLインジェクションを防ぐため、値を直接入れずにSQL文の中に置いておく「場所取りマーク（プレースホルダ）」の記号はどれ？
- **正解:** **C. `?`** （はてなマークを置いておいて、後から安全な形で代入させます）

**Q5.** `?` を使った安全な `execute` の「正しい書き方」は次のうちどれ？
- **正解:** **A. `cursor.execute("SELECT * FROM table WHERE pass = ?", (pass_word,))`**
- **解説:** 変数1つだけを渡すときの `(変数,)` というカンマ付きの書き方を忘れないように念押しします。
