## 👨‍🏫 メンタリング（コーチング）の基本スタンス

### 1. エラーが出たと呼ばれたら
- **絶対にやってはいけないこと**: 先生がキーボードを奪って修正し、「はい直ったよ」と立ち去る。
- **正解のステップ**:
  1. 「何しようとしたら、どうなっちゃったの？」と**状況を言語化（説明）**させる。
  2. 「下に出てる赤い文字（エラーメッセージ）、なんて読める？」と**エラーを直視**させる。
  3. 「student.mdのエラー辞典のどこに当てはまるかな？」と一緒に探す。
  4. 「あ、カッコが足りないんだな。どこだろう？この行の前後を見てごらん」と**場所だけ**を教え、本人にタイピングさせる。

### 2. 「〇〇という機能を作りたい！」への対応
- 例：「先生、パスワードを入力しないと開かないようにしたいです！」
- **正解のステップ**:
  1. 「お、いいね！」とまずは肯定する。
  2. 「どういう手順（アルゴリズム）なら実現できそうかな？」と分解させる。
     - *（生徒：ええと、画面にパスワード入力欄を作って…）*
     - *（生徒：ボタンを押したら、DBの中身と合ってるか調べる？）*
  3. 「大正解！じゃあ、①入力欄を作る処理、②ボタンの関数、③SELECTでWHERE password = ? で探す処理。この3ステップに分けて、まずは①から書いてみよう」と、**タスクを細分化**してあげます。

## 🛠️ 教員特権：頻繁に発生する「ハマりポイント」と救済措置

この3週間で、ほぼ確実にあちこちで発生する「沼」について事前にお伝えしておきます。

### 沼 1：`database is locked` （DBロックエラー）
- **現象**: プログラムは動いているのに、ボタンを押すと `database is locked` と出て固まる。
- **原因**: 別の画面やツール（SQLite Viewerなど）、あるいは先ほどの関数で `connection.close()` をし忘れて「電話をかけっぱなし（掴みっぱなし）」になっているため、次の電話が繋がらずロックされています。
- **解決策**: 「いま開いてるPythonプログラムやVS Codeのタブを、一旦『全部』×ボタンで閉じてもう一回実行してごらん」と指示します。大体これで直ります。

### 沼 2：`object has no attribute` （部品の綴り間違い等）
- **現象**: `AttributeError: 'Button' object has no attribute 'packk'` などと言われる。
- **原因**: シンプルなスペルミスです（`pack()` を `packk()` にしている等）。
- **解決策**: Pythonは英語の大文字小文字に極度に厳しい言語です（`tkinter` と `Tkinter` も別物）。エラーメッセージの最後にある綴りと、教科書の綴りを見比べさせてください。

### 沼 3：無限ループから抜け出せない
- **現象**: `while True:` などで作った処理の中でエラーメッセージが無限に出続けて、パソコンがフリーズしそうになる。
- **原因**: プログラムが制御不能に陥っています。
- **解決策**: 「ショートカットキー **Ctrl + C** を押せ！」と叫んでください。これは強制終了（Kill）の絶対コマンドです。
