## レッスン 1: 組織の3層構造

企業の組織は、役割に応じて大きく **3つの層** に分かれています。

```
┌─────────────────────────────┐
│        経営者層              │  ← 少人数・会社全体の方向を決める
├─────────────────────────────┤
│        管理者層              │  ← 中間管理職・部門をまとめる
├─────────────────────────────┤
│      一般従業員層            │  ← 現場の実務を担当する
└─────────────────────────────┘
```

| 層 | 主な役職例 | 役割 |
|----|-----------|------|
| **経営者層** | 社長・副社長・取締役 | 会社全体の経営方針・戦略を決定する |
| **管理者層** | 部長・課長・チームリーダー | 経営方針を受けて部門・チームを管理する |
| **一般従業員** | 一般社員・スタッフ | 各部門の実際の業務を担当する |

## レッスン 2: 組織の分け方

会社の業務をどの単位で分けるかによって、組織の形が変わります。代表的な2つを学びましょう。

### 職能別組織

> 「何をするか（機能）」で部門を分ける組織形態です。

```
          【社長】
     ┌──────┼──────┐
  営業部   製造部   経理部
```

| 特徴 | 内容 |
|------|------|
| 分け方 | 仕事の機能・種類ごと（営業・製造・経理など） |
| メリット | 専門性が高まる・同じ職種が集まるので効率的 |
| デメリット | 部門間の連携が取りにくくなりやすい |
| 向いている場面 | 商品・サービスの種類が少ない企業 |

### 事業部制組織

> 「何を扱うか（製品・地域・顧客）」で部門を分ける組織形態です。

```
          【社長】
     ┌──────┼──────┐
  家電事業部  PC事業部  スマホ事業部
  (営業・製造・  (営業・製造・  (営業・製造・
   経理が一体)   経理が一体)   経理が一体)
```

| 特徴 | 内容 |
|------|------|
| 分け方 | 製品・地域・顧客ごとに独立した事業単位 |
| メリット | 事業ごとの意思決定が速い・責任が明確 |
| デメリット | 各事業部に機能が重複し、コストが増えやすい |
| 向いている場面 | 複数の製品・市場を扱う大企業 |

## レッスン 3: 組織がうまく回るための3原則

どんな組織形態でも、うまく機能させるには共通のルールが必要です。

### ① 命令系統一元化の原則

> **1人の部下は、1人の上司からのみ命令を受ける**というルールです。

```
❌ 悪い例：上司が2人いて、命令が矛盾する
   部長A「急いでAを仕上げて！」
   部長B「いや、先にBを！」
   → 部下が混乱する

✅ 良い例：命令は1人の上司から
   課長「今日はAを優先してください」
   → 部下が迷わず動ける
```

### ② 権限と責任一致の原則

> **与えられた責任に見合った権限も与える**というルールです。

```
❌ 悪い例：責任だけあって権限がない
   「この案件、君が責任者だ。でも決済は全部上司の承認が必要」
   → 責任者なのに何も決められない

✅ 良い例：責任と権限がセット
   「この案件の責任者として、50万円までは君が決済できる」
   → 迅速に判断・行動できる
```

### ③ 統制の範囲の原則

> **1人の管理者が直接管理できる部下の人数には限界がある**というルールです。

一般的に、1人の管理者が適切に管理できる人数は **5〜10人程度** といわれています。

```
❌ 悪い例：1人が50人を管理
   → 一人ひとりへの指示・確認が行き届かない

✅ 良い例：1人が7人を管理
   → 十分なコミュニケーションと管理が可能
```

## レッスン 4: 会社の意思決定の仕組み

会社の重要な決定はどのように行われるのでしょうか？

### 株式会社の意思決定フロー

```
【株主総会】
  株主が集まり、会社の重要事項を決議する（最高の意思決定機関）
       ↓ 取締役を選任
【取締役会】
  取締役が集まり、経営の具体的な方針・重要事項を決定する
       ↓ 業務執行の指示
【代表取締役（社長）・各部門】
  日常の経営・業務の執行を行う
```

| 機関 | 構成メンバー | 決めること |
|------|-------------|-----------|
| **株主総会** | 株主（出資者） | 会社の基本方針・取締役の選任・配当など |
| **取締役会** | 取締役（経営陣） | 経営の方針・重要な業務執行の決定 |
| **代表取締役** | 社長など | 日常の業務執行・対外的な代表 |

### 日本特有のシステム「稟議（りんぎ）」

> **稟議**とは、業務上の重要な判断や決定を、担当者から上位の決裁者へ順番に承認を求めながら進める、日本特有の意思決定の仕組みです。

```
【稟議の流れ】

担当者が稟議書を作成
       ↓
係長・課長が確認・承認
       ↓
部長が確認・承認
       ↓
役員・社長が最終承認
       ↓
正式に決定・実行
```

| 稟議の特徴 | 内容 |
|-----------|------|
| **メリット** | 多くの人が確認するため、ミスや偏った判断を防げる |
| **メリット** | 関係者全員が情報を共有できる |
| **デメリット** | 承認に時間がかかり、意思決定が遅くなる |
| **デメリット** | 誰が最終的に責任を持つか曖昧になりやすい |

欧米企業では上司が単独で迅速に決定するスタイルが多いのに対し、日本では**組織全体の合意を重視する**稟議スタイルが根付いています。

## ドリル 1-2: 3原則を適用しよう

次の状況は、3原則のどれに違反しているか答えましょう。

```
（  ）① 山田さんは「この件の責任者」と言われたが、
        支出の決済権限は一切与えられていない

（  ）② 鈴木部長は100人の部下を1人で直接管理している

（  ）③ 田中さんは、A課長からもB課長からも別々の指示を受けている
```

<details>
<summary>答えを見る</summary>

- ① 権限と責任一致の原則（責任はあるが権限がない）
- ② 統制の範囲の原則（1人が管理できる人数を超えている）
- ③ 命令系統一元化の原則（2人の上司から指示を受けている）

</details>
