- **【エラー体験】**: ドリル「エラーを体験しよう！」でわざと `" "` を消させます。

### 2️⃣ ドリル演習と全角スペースハンティング (30分〜60分)

- **【演習】**: 「ドリル 1-1」をやらせます。
- ここで、日本人のプログラミング学習の最大の敵「**全角スペース**」が猛威を振るいます。
  - `print("あいう"　)` のように全角スペースが混ざると `SyntaxError` が出ます。
  - 「赤い波線が出たら、先生を呼ぶ前に全角スペースがないか探してごらん！」と自力での解決を促します。
- **【FAQ】**:
  - `Q. コメント（#）の行は何のためにあるの？` → `A. パソコンは読まない「人間のためのメモ」だよ。コードを分かりやすくするために使います。`

### 3️⃣ 変数という「箱」の概念 (60分〜100分) ※間で休憩10分

- **【解説】**: 黒板に「ダンボール箱」の絵を描き、側面に名札（変数名）、中に果物など（データ）を書きます。「箱がないと、データを次の命令で使いまわせないよ」と必要性から入ってください。
- **【演習】**: 「レッスン2」および「ドリル2-1」をやらせます。
- **【FAQ】**:
  - `Q. 箱の名前は何でもいいの？` → `A. 日本語でも動くけど、世界共通ルールの英語（アルファベット）を使うのがプロのやり方だよ、と指導してください。`
  - `Q. print("name") と print(name) の違いは？` → `A. "name" はただの文字のシール。name（クォートなし）は「nameという名前の箱を見てね」という意味。決定的に違います。`

### 4️⃣ f文字列と自己紹介ツール (100分〜140分)

- **【解説】**: f文字列を使わないと `print("私の名前は" + name + "です")` のように面倒であることを実際に書いて見せ、f文字列の便利さを実感させます。
- **【演習】**: 「レッスン3」および「ドリル3-1」をやらせます。
- ⚠️ **超頻出ミス**: `{ }` を `[ ]` や `( )` と間違える生徒が多いです。「波カッコだよ！」と黒板に大きく書いておいてください。
- ⚠️ **もう一つの頻出ミス**: `f` の位置を間違える（`print("f...")`など）生徒が必ずいます。「fはクォーテーションの外、左側につけるよ」と強調してください。

## 📝 小テストの解答と解説（全5問）

**Q1.** パソコンの画面に文字を表示させるための命令はどれ？

- **正解:** **B. `print()`**
- **解説:** `show()` や `output()` はPythonに存在しない命令です。

**Q2.** 文字を表示させるとき、正しく囲まれているのはどれ？

- **正解:** **C. `print('こんにちは')`**
- **解説:** シングルまたはダブルクォーテーションで囲むのが正解。囲まないと変数として扱われ `NameError` になります。

**Q3.** プログラミングにおける `=`（イコール1つ）の正しい意味はどれ？

- **正解:** **B. 右のものを、左の箱に入れる（代入する）という意味**
- **解説:** 数学の「等しい」と混同させないことが大切です。比較には `==` を使います（来週以降）。

**Q4.** 変数 `item` の中身を表示させる正しい書き方はどれ？

- **正解:** **B. `print(item)`**
- **解説:** `print("item")` だと、ただの「i・t・e・m」という4文字が表示されます。クォートの有無で意味がまったく変わることを強調してください。

**Q5.** 変数の中身を文章に埋め込む「f文字列」の正しい書き方はどれ？

- **正解:** **A. `print(f"これは {item} です")`**
- **解説:** `f` の位置はクォーテーションの外（左側）、変数を埋め込む場所は波カッコ `{ }` を使うのが大鉄則です。
