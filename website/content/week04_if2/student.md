## ステップ1: `elif`（エリフ）の登場

「もし80点以上ならA、**そうじゃなくて**もし60点以上ならB、それ以外ならC」というように、複数の条件を順番にチェックしたいときは **`elif`** を使います。
※「else if」の略です。

### 👉 やってみよう (1)
遊園地のアトラクションの「身長制限チェッカー」を作ってみましょう。

```python
height = int(input("身長は何cmですか？: "))

if height >= 150:
    print("大迫力の絶叫コースターに乗れます！")
elif height >= 120:
    # 150cmは無いけど、120cm以上はある場合
    print("ワクワクの急流すべりに乗れます！")
else:
    # どっちにも当てはまらない（120cm未満）場合
    print("メリーゴーランドに乗ろう！")
```

実行して、自分の身長を入力してみよう。どの乗り物に乗れたかな？

## ステップ2: 条件を見極める順番が命！

`if`, `elif`を使ったプログラムには**「上から順番にチェックされる」**という絶対ルールがあります。

### 👉 やってみよう (2)
わざと順番を間違えた「ダメなプログラム」を実行してみましょう。

```python
score = int(input("テストの点数は？: "))

# あれ？順番がおかしいぞ？
if score >= 0:
    print("赤点です。がんばりましょう！")
elif score >= 80:
    print("合格です！素晴らしい！")
```

大天才のあなたは「100」を入力しました。さて、結果はどうなったでしょうか？

**【結果の確認】**
100点なのに「赤点です」と表示されてしまいましたね！
なぜなら、プログラムは上から「100は0以上かな？…Trueだ！じゃあ赤点を表示してここで終わり！」と判断してしまったからです。
**狭い（厳しい）条件から順番に書く**のがプロの鉄則です。

## ステップ3: デーモンを倒すRPGを作ろう！

ここまでの知識（入力・計算・条件分岐）を使えば、簡単なRPGの1シーンを作ることができます。

### 👉 やってみよう (3)
主人公の攻撃力を入力して、モンスターを倒せるか判定するプログラムです。

```python
print("野生のデーモンが現れた！(HP: 100)")
attack = int(input("あなたの攻撃力はいくつ？(1〜200): "))

# 攻撃力が届かなければ負け、ピッタリならギリギリ勝利、オーバーなら圧勝
if attack >= 150:
    print("会心の一撃！！デーモンを粉砕した！！")
elif attack >= 100:
    print("激闘の末、デーモンを倒した！")
else:
    print("攻撃が効かない！あなたは逃げ出した…")
```

自分で数字を変えて、すべてのメッセージを表示させることができるか挑戦してみましょう！
