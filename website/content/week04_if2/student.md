## 🕒 レッスン 1: 「そうじゃないなら」の `else` と `elif`

`if` に続けて、条件に合わなかったときの別の行動を指定できるのが **`else`（エルス）** と **`elif`（エルイフ：else ifの略）** です。

### 💻 ハンズオン（やってみよう）
年齢によって料金が変わる映画館のプログラムです。実行して、いろんな年齢を入力してみましょう。

```python
age = int(input("年齢を入力してください: "))

if age < 12:
    print("こども料金：1000円です。")
elif age < 18:
    print("中高生料金：1500円です。")
else:
    print("大人料金：1800円です。")
```

> **💡 ルール**
> 1. 最初は必ず `if` から始まります。
> 2. 間の条件はいくつでも `elif` を追加できます。
> 3. 最後の「それ以外すべて」は `else:` になります（`else` には条件式を書きません）。

## 🕒 レッスン 2: 魔法の接着剤 `and` と `or`

「身長が120cm以上」**かつ**「年齢が10歳以上」というように、複数の条件をくっつける接着剤が **`and`** と **`or`** です。

- **`and`** : 両方ともTrueのときだけTrueになる（厳しい！）
- **`or`** : どちらか片方でもTrueならTrueになる（優しい！）

### 💻 ハンズオン（やってみよう）
以下のコードを実行し、ローラーコースターに乗れるかテストしてみましょう。

```python
height = int(input("身長(cm)を入力: "))
age = int(input("年齢を入力: "))

# 身長が130以上 「かつ」 年齢が10歳以上なら乗れる！
if height >= 130 and age >= 10:
    print("絶叫ローラーコースターに乗れます！楽しんでね！")
else:
    print("ごめんなさい、安全のため乗れません。")
```

## 🕒 レッスン 3: 順番のワナ（論理崩壊のパズル）

`if` 〜 `elif` のブロックは、**「上から順番に判定され、一度 True になったら下の条件はもう無視される」** という超重要ルールがあります。

### 💻 手品（バグ）の体験
このプログラムは間違っています。実行して「100点」を入力してみてください。

```python
score = int(input("点数は？: "))

if score > 0:
    print("もう少しがんばりましょう。")
elif score > 50:
    print("よくできました！")
elif score > 80:
    print("たいへんよくできました！！")
else:
    print("0点ですね。")
```

「100点」を入れたのに「もう少しがんばりましょう。」と表示されませんでしたか？
これは、一番上の `score > 0`（0点より大きい）の条件に引っかかって、そこで終わって（下を無視されて）しまったからです！

### ✍️ ドリル 3-1：順番パズル
上記のコードの条件の **順番を並べ替えて**、100点の人が正しく「たいへんよくできました！！」と表示されるように直してください。
（ヒント：厳しい条件（数字が大きいもの）から順番に上から書くのがコツです！）
