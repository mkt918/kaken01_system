## 🕒 レッスン 1: 「そうじゃないなら」の `else` と `elif`

`if` に続けて、条件に合わなかったときの別の行動を指定できるのが **`else`（エルス）** と **`elif`（エルイフ：else if の略）** です。

### 💻 ハンズオン（やってみよう）

年齢によって料金が変わる映画館のプログラムです。実行して、いろんな年齢を入力してみましょう。

```python
age = int(input("年齢を入力してください: "))

if age < 12:
    print("こども料金：1000円です。")
elif age < 18:
    print("中高生料金：1500円です。")
else:
    print("大人料金：1800円です。")
```

> **💡 ルール**
>
> 1. 最初は必ず `if` から始まります。
> 2. 間の条件はいくつでも `elif` を追加できます。
> 3. 最後の「それ以外すべて」は `else:` になります（`else` には条件式を書きません）。

## 🕒 レッスン 2: 魔法の接着剤 `and` と `or`

「身長が120cm以上」**かつ**「年齢が10歳以上」というように、複数の条件をくっつける接着剤が **`and`** と **`or`** です。

- **`and`** : 両方ともTrueのときだけTrueになる（厳しい！）
- **`or`** : どちらか片方でもTrueならTrueになる（優しい！）

### 💻 ハンズオン（and でローラーコースター）

以下のコードを実行し、ローラーコースターに乗れるかテストしてみましょう。

```python
height = int(input("身長(cm)を入力: "))
age = int(input("年齢を入力: "))

# 身長が130以上 「かつ」 年齢が10歳以上なら乗れる！
if height >= 130 and age >= 10:
    print("絶叫ローラーコースターに乗れます！楽しんでね！")
else:
    print("ごめんなさい、安全のため乗れません。")
```

### ✍️ ドリル 2-1：優しい or の体験

会員カードを持っているか、または（or）シニア（60歳以上）であれば割引になるスーパーのプログラムです。
`or` を使って条件式を完成させてください。

```python
print("※カードを持っている場合は 1、持ってない場合は 0 を入れてね")
has_card = int(input("カードは持ってる？: "))
age = int(input("年齢は？: "))

# has_card が 1 と等しいか、または、age が 60 以上なら割引
if has_card == 1 or age >= 60:    # ← この条件式を自分で考えて書いてみよう！
    print("10%割引になります！")
else:
    print("通常料金です。")
```

> **⚠️ よくあるミス**
> `if has_card == 1 or 60:` のように右側の比較を省略することはできません。
> パソコンは毎回「何と比べるか」を明示してあげる必要があります。
> 正解は `if has_card == 1 or age >= 60:` です。

## 🕒 レッスン 3: 順番のワナ（論理崩壊のバグ）

`if` 〜 `elif` のブロックは、**「上から順番に判定され、一度 True になったら下の条件はもう無視される」** という超重要ルールがあります。

### 💻 バグを体験しよう

このプログラムは間違っています。実行して「100」を入力してみてください。

```python
score = int(input("点数は？: "))

if score > 0:
    print("もう少しがんばりましょう。")
elif score > 50:
    print("よくできました！")
elif score > 80:
    print("たいへんよくできました！！")
else:
    print("0点ですね。")
```

「100」を入れたのに「もう少しがんばりましょう。」と表示されましたね？
これは、一番上の `score > 0`（0点より大きい）の条件に最初にひっかかって、そこで判定が終わってしまったからです！

### ✍️ ドリル 3-1：順番パズル

上記のコードの条件の **順番を並べ替えて**、100点の人が正しく「たいへんよくできました！！」と表示されるように直してください。

> **ヒント:** 厳しい条件（数字が大きいもの）から順番に上から書くのがコツです！
