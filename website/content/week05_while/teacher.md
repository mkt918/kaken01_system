# 指導案: Week 05 「終わりなき繰り返し（while）」

## 🎯 今週のねらいと授業のゴール
プログラミングの醍醐味である「ループ（繰り返し）」を習得します。
最大のハイライトは「無限ループの恐ろしさを実際に体験させる」ことと、「強制終了（Ctrl+C）のやり方を体に刻み込ませる」ことです。

---

## ⏱️ タイムテーブル (180分)

### 1️⃣ 導入と while ループ (0分〜40分)
- **【解説】**: コンピュータが人間より優れている唯一の点は、「文句を言わず何千回でも同じことを速く繰り返せることだ」と伝えます。
- **【演習】**: 「レッスン1」を実行させます。
- **【指導のポイント】**:
  - `count = count + 1` という記述について、「数学の先生が見たら怒る式だけど、プログラミングでは『右で計算して、左の箱に上書き保存する』という超重要構文なんだよ」と黒板で丁寧に図解してください。これを理解しないと今後のすべてで躓きます。
  - 「ドリル1-1」でカウントダウンを作らせます。

### 2️⃣ 恐怖の無限ループ体験イベント (40分〜80分)
- **【解説】**: ループには絶対に破ってはいけないルール（変化を書かないと終わらない）があることを語ります。
- **【演習】**: 「レッスン2」を、あえて全員一斉に実行させます。
- **【大パニックのコントロール】**:
  - 画面が猛烈に流れ続ける「無限ループ」に、生徒は「壊れた！」「止まらない！」とパニックになります。
  - そこで冷静に「**Ctrl と C を同時に押せ！**」と指示を出します。
  - Macの生徒がいれば「Control と C だ！」と個別に叫んでください。
  - この「Ctrl+Cで強制終了できる」という安心感を与えてから、プログラミングを再開させます。

### 3️⃣ break と無限ループの正しい使い方 (80分〜120分) ※間で休憩10分
- **【解説】**: 一見悪者の無限ループも、「自らブレーキ（`break`）を踏めるなら」最強のテクニックになることを解説します。
- **【演習】**: 「レッスン3」のチャットボットと、「ドリル3-1」のパスワード認証をやらせます。
- **【トラブルシューティング】**: 
  - `break` を `if` の中（インデントを下げる）に入れないと、1回目で無条件にループが終わってしまいます。インデントの重要性を再確認させてください。

### 4️⃣ 総仕上げ：ダブル・ミニプロジェクト (120分〜170分)
- **【総合演習 1】**: 前回の改良版、「当たるまで終わらない数当てゲーム」を作ります。
  - 余裕があれば、「入力した数字が、正解より『大きいか』『小さいか』をヒントで出してみよう」とアレンジを提案すると熱中します。
- **【総合演習 2】**: 生徒が大好きな「RPGの戦闘シーン」です。
  - `while slime_hp > 0:` の条件文を書けるかが勝負です。
  - 「逃げる（`break`）」を忘れると、永遠に殴られ続ける（または永遠にHPがマイナスになり続ける）バグが発生します。うまく無限ループに陥った生徒をイジりつつ指導してください。

### 5️⃣ 小テストと振り返り (170分〜180分)
- 終わりの10分で小テストを実施し、答え合わせをして解散します。

## 📝 小テストの解答と解説（全5問）

**Q1.** プログラムで「〜の間、繰り返す」という命令はどれ？
- **正解:** **C. `while`**

**Q2.** プログラミングにおける `count = count + 1` の意味として正しいのは？
- **正解:** **B. 今の箱の中身に1を足して、また同じ箱に戻す（カウントアップ）**

**Q3.** 終わらない繰り返し（無限ループ）から強制的に脱出（停止）させるキーボード操作はどれ？
- **正解:** **A. `Ctrl + C`**
- **解説:** このショートカットは、ターミナルを使う世界中のエンジニアが毎日使う必須スキルです。

**Q4.** 繰り返し（ループ）を、途中で強制的に抜け出すための「ブレーキ」となる命令は？
- **正解:** **C. `break`**

**Q5.** わざと無限ループを作りたいときによく使う、永遠に繰り返すための書き方はどれ？
- **正解:** **B. `while True:`**
- **解説:** 条件が True（常に正しい）なので、永遠にお部屋（ループ）の中に閉じ込められます。
