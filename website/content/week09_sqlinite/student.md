## 🕒 レッスン 1: データベースとは？Excelとの違い

データベースは、ExcelやGoogleスプレッドシートによく似た「表（テーブル）」の形をしています。
しかし、Excelは基本的に「1人が1つの画面を開いて編集する」のに対し、データベースは**「何千人、何万人ものユーザーが同時に高速にデータを読み書きする」**ための特殊なシステムです。

### 📊 データベースの基本用語（絶対に覚えよう！）

- **テーブル (Table)**: データのまとまり（例：「勇者テーブル」「モンスターテーブル」）。Excelの1シートのようなもの。
- **カラム (Column/列)**: 縦の項目（例：「名前」「レベル」「HP」）。
- **レコード (Record/行)**: 横のデータ1件分（例：「アーサー, 10, 500」という勇者1人分のデータ）。

> **💡 例えると…**
> Excelで「クラスの名簿」を作ったとします。
> そのシート全体が **テーブル**、「名前」「出席番号」などの列タイトルが **カラム**、1人分の行データが **レコード** です！

## 🕒 レッスン 2: 最初のテーブルを作ってみよう（CREATE TABLE）

箱（変数）を作るときにPythonでは `name = "たろう"` と書きましたが、SQLでは「どんな名前の列（カラム）を、どんなデータ型で作るか」をキッチリ宣言する必要があります。

### 💻 ハンズオン（オンラインSQLで実行しよう！）

先生が指定した「SQLを練習できるウェブサイト（sqliteonline.comなど）」を開き、左側の入力欄に以下の魔法の呪文（SQL）を書いて実行（Run）してください。

```sql
/* 生徒（students）のデータを保存するテーブルを作る */
CREATE TABLE students (
    id INTEGER,      /* 数字を入れる列 */
    name TEXT,       /* 文字を入れる列 */
    grade INTEGER    /* 学年（数字）を入れる列 */
);
```

> **💡 解説**
>
> - `CREATE TABLE` は「テーブルを作れ！」という命令です。
> - `INTEGER`（インティジャー）は「整数・数字」、`TEXT`（テキスト）は「文字」という意味です。データベースは型（文字か数字か）にとても厳しいです！
> - 一番おしりには必ず `;`（セミコロン）をつけます。「ここで呪文はおしまい」の合図です。

## 🕒 レッスン 3: 倉庫にデータを投げ込む（INSERT）

テーブルという「枠」ができたので、実際にデータ（レコード）を入れていきましょう！
データを入れる魔法は `INSERT INTO`（インサート・イントゥ）です。

### 💻 ハンズオン（やってみよう）

以下のSQLを実行して、さきほど作った `students` テーブルに3人のデータを入れてみましょう。

```sql
/* INTO テーブル名 (カラム名...) VALUES (入れる中身...); */
INSERT INTO students (id, name, grade) VALUES (1, 'たろう', 1);
INSERT INTO students (id, name, grade) VALUES (2, 'はなこ', 2);
INSERT INTO students (id, name, grade) VALUES (3, 'さぶろう', 1);

/* 中身がどうなったか「全部見せて！」の呪文 */
SELECT * FROM students;
```

> **💡 解説と重要ルール**
>
> - `INSERT INTO` のあとにテーブル名を書き、そのあとに `(列1, 列2...) VALUES (データ1, データ2...)` と書きます。順番がズレてはいけません！
> - ⚠️ **文字データは必ず `' '`（シングルクォーテーション）で囲む！** `"`（ダブル）ではなく `'`（シングル）を使うのがSQLの絶対ルールです。Pythonとは少し違いますね。

### ✍️ ドリル 3-1：勇者を追加しよう

ドリル2-1で作った `heroes` テーブルに、以下のデータを入れて確認してください。

```sql
INSERT INTO heroes (id, name, hp) VALUES (1, 'アーサー', 100);
INSERT INTO heroes (id, name, hp) VALUES (2, 'ランスロット', 80);

/* 全部見せて！の呪文 */
SELECT * FROM heroes;
```

> **確認：** `'アーサー'` のようにシングルクォーテーションで囲めていますか？忘れるとエラーになります！
