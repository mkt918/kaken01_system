## ⏱️ タイムテーブル (180分)

### 1️⃣ 導入：なぜデータベースなのか？ (0分〜30分)
- **【解説】**: 「Pythonで作ったゲーム、電源切ったらレベルが1に戻っちゃったよね？」「モンストやパズドラが毎回レベル1に戻ったら発狂するでしょ？」と身近な例を出し、「永遠にデータを保存する魔法の倉庫」の必要性を説きます。
- **【用語の徹底】**: 「テーブル・カラム・レコード」というIT業界の共通言語をここで覚えさせます。Excelを見せながら「これがテーブル、縦がカラム、横の一行がレコード！」と視覚的に叩き込みます。

### 2️⃣ 倉庫の箱作り（CREATE TABLE） (30分〜60分)
- **【解説】**: DBは几帳面。最初に「何の列があるか、データは数字(INTEGER)か文字(TEXT)か」を宣言しないと怒られることを教えます。
- **【演習】**: 使用するオンラインSQLツールを開かせ、「レッスン2」のコードをタイピング（またはコピー）して実行（Run）させます。
- **【トラブルシューティング】**: 
  - `INTEGER` のスペルミス（INTERGERなど）が多発します。「整数」という意味の英単語だと教え、ゆっくり打たせます。
  - 最後のセミコロン `;` を忘れるとSQLは動かないことを強調します。

### 3️⃣ データを投げ込む（INSERT） (60分〜100分) ※間で休憩10分
- **【解説】**: 枠組みにドカドカとデータを入れる `INSERT INTO` をやります。
- **【演習】**: 「レッスン3」から一気にエラーが激増します。
- **【頻出エラーと指導の声かけ】**:
  - `INSERT INTO students (id, name, grade) VALUES (1, たろう, 1);` ➞ エラー！「たろうは人間（文字）だから、`'`（シングルクォーテーション）という服を着せないとDBの世界には入れないよ！」と教えます。
  - カンマ `,` をドット `.` と打ち間違える生徒がいます。「ドットはダメ！息継ぎのカンマだよ」と見回ってください。

### 4️⃣ 総仕上げ：ダブル・ミニプロジェクト (100分〜170分)
- SQLの真の面白さは「自分が作ったデータが綺麗な表になって出てくる」ことです。
- **【総合演習 1】**: 「部活名簿」。`CREATE` と `INSERT` と `SELECT *` を一気通貫でやらせます。
- **【総合演習 2】**: 「RPGモンスター図鑑」。
  - 項目が4つに増えるため、カッコの中身のカンマの対応が複雑になります。
  - 「`id` に文字を入れてみるとどうなる？」「わざと怒られてみよう」と、あえてエラーを出してエラーメッセージを読む練習をさせると良いです。
  - 5件登録できた生徒には、「10件登録して、HPを1万とかに設定して最強の図鑑を作ってみて！」と促します。

## 📝 小テストの解答と解説（全5問）

**Q1.** データベースにおいて、「名前」「レベル」「HP」など、縦の項目のことを何と呼ぶ？
- **正解:** **C. カラム (Column/列)**
- **解説:** 横の1件のデータは「レコード (Record)」です。必ず暗記させましょう。

**Q2.** SQLで、新しいテーブル（表）の枠組みを作るための命令（呪文）はどれ？
- **正解:** **B. `CREATE TABLE`**
- **解説:** 英語の「Create（作る）」とそのままです。

**Q3.** SQLで「文字」のデータを保存するとき、絶対に付けなければいけない記号はどれ？
- **正解:** **B. `'` (シングルクォーテーション)**
- **解説:** Pythonでは `""` も使えましたが、SQL言語の標準ルールでは文字は `' '` （シングル）で囲むという厳密なルールがあります。

**Q4.** SQLで、テーブルに新しいデータ（行）を追加する（投げ込む）ための命令は？
- **正解:** **A. `INSERT INTO`**
- **解説:** インサート（挿入する）＝データを入れる、という意味です。

**Q5.** SQL言語で、一つの命令文（呪文）が「ここで終わり」であることを示すために最後につける記号はどれ？
- **正解:** **C. `;` （セミコロン）**
- **解説:** ピリオド（.）でもなく、コロン（:）でもありません。MySQLやPostgreSQLなど、プロの現場でもセミコロンが必須のデータベースは多いので、初心者のうちからクセをつけさせます。
