## 🕒 レッスン 2: 魔法のフィルター `WHERE`（条件絞り込み）

データが何万件もあるとき、上から全部見るのは不可能です。
「HPが100以上のやつだけ出ろ！」と条件をつける魔法が **`WHERE`（ウェア）** です。Pythonの `if` と似ていますね。

### 💻 ハンズオン（やってみよう）
```sql
/* HPが「100以上 (>=)」のモンスターだけを抽出する */
SELECT name, hp 
FROM monsters 
WHERE hp >= 100;
```

> **💡 解説: SQLの比較記号**
> - `>=` : 以上
> - `<=` : 以下
> - `>` : より大きい
> - `<` : より小さい
> - `=` : 同じ（Pythonは `==` でしたが、SQLは `=` を1つだけ使います！）

### ✍️ ドリル 2-1：属性で絞り込む
`type` が `'水'`（シングルクォーテーションを忘れずに！）のモンスターだけを抽出して、名前と属性を表示してください。

```sql
SELECT name, type 
FROM monsters 
WHERE type = _________;
```

## 🕒 レッスン 3: 順番に並べ替える `ORDER BY`

ゲームの「スコアランキング」と同じように、数字が大きい順や小さい順に並べ替える魔法が **`ORDER BY`（オーダーバイ）** です。

### 💻 ハンズオン（やってみよう）
```sql
/* HPが「大きい順」に並べ替える (DESC = 降順/大きい方が上) */
SELECT name, hp 
FROM monsters 
ORDER BY hp DESC;

/* HPが「小さい順」に並べ替える (ASC = 昇順/小さい方が上) */
SELECT name, hp 
FROM monsters 
ORDER BY hp ASC;
```

## 🚀 今日の総仕上げ2：ミニプロジェクト「スパイを探せ！条件検索ゲーム」

巨大な `citizens`（市民）データベースの中に、たった1人の「重大なスパイ」が紛れ込んでいます！
警察からの情報（条件）を元に、スパイが誰なのかを特定してください。
※条件は `AND` という言葉で繋ぐと「Aであり、かつBである」という複数条件にできます。

### 📝 ミッション
**警察からの極秘情報：**
- スパイの年齢 (`age`) は **30歳以上** である。
- スパイの性別 (`gender`) は **'男'** である。
- スパイの身長 (`height`) は **175より大きい**。

**【クエリ（SQL）のヒント】**
```sql
SELECT * 
FROM citizens 
WHERE age >= 30 
  AND gender = '男' 
  AND ___________;
```

最後に絞り込まれて残った1人が犯人です。犯人の名前を当ててください！
