## 🕒 レッスン 1: 曖昧（あいまい）な検索 `LIKE` の魔法

「一部だけ一致していればOK！」という検索には、`=` の代わりに **`LIKE`** を使い、さらに **`%`（パーセント）** という「何かの文字が入るよマーク（ワイルドカード）」を組み合わせます。

### 💻 ハンズオン（やってみよう）
先生が用意した事前準備コードをRunして、大量の敵データ（`enemies` テーブル）を準備してください。

```sql
/* 「ド」から始まる名前の敵を全て探す */
SELECT * FROM enemies
WHERE name LIKE 'ド%';

/* どこかに「ドラゴン」を含む敵を全て探す（前後どちらに文字があってもOK） */
SELECT * FROM enemies
WHERE name LIKE '%ドラゴン%';
```

> **💡 解説: `%` の位置が命！**
> - `'ド%'`: 前方一致（「ド」で始まる。ドラキー、ドラゴンなど）
> - `'%ス'`: 後方一致（「ス」で終わる。スライムは×。ボストロールスなら○）
> - `'%水%'`: 部分一致（どこに入っていてもOK）

### ✍️ ドリル 1-1：レッドを探せ！
名前に `'レッド'` という言葉を含むモンスターの全てのデータを抽出してください。

```sql

WHERE name ________ '%レッド%';
```

## 🕒 レッスン 2: トップだけを取り出す `LIMIT`

100万人のゲームプレイヤーがいるとき、ランキングを見るために全員のデータを表示していたら画面がいっぱいになってしまいます。
「上位○件だけを見せて！」という便利魔法が **`LIMIT`（リミット）** です。

### 💻 ハンズオン（やってみよう）
先週習った `ORDER BY`（並べ替え）と組み合わせるのが最強のテクニックです。

```sql
/* HPが高い順（DESC）に並べ替えて、上位『3件だけ』を表示する */
SELECT name, hp 
FROM enemies
ORDER BY hp DESC
LIMIT 3;
```

### ✍️ ドリル 2-1：最弱モンスター決定戦
`enemies` テーブルの中で、HPが一番 **低い** モンスターの「名前」と「HP」を **1匹だけ** 表示してください。（`ASC` と `LIMIT` のコンボを使います）

```sql
SELECT name, hp
FROM enemies
ORDER BY hp _______
LIMIT _______;
```

## 🕒 レッスン 3: カブリを消し去る `DISTINCT`

たくさんデータがあるとき、「全部で何種類の『属性（type）』があるんだろう？」と知りたいことがあります。
同じものをギュッと1つにまとめて（重複を排除して）表示する魔法が **`DISTINCT`（ディスティンクト）** です。

### 💻 ハンズオン（やってみよう）
```sql
/* 普通に属性を出すと、炎、炎、水、水、炎... と大量に出てしまう */
SELECT type FROM enemies;

/* DISTINCT を使うと、かぶりを消して「炎、水」など種類だけを教えてくれる！ */
SELECT DISTINCT type FROM enemies;
```
