## 🕒 レッスン 1: 曖昧（あいまい）な検索 `LIKE` の魔法

「一部だけ一致していればOK！」という検索には、`=` の代わりに **`LIKE`** を使い、さらに **`%`（パーセント）** という「何かの文字が入るよマーク（ワイルドカード）」を組み合わせます。

### 💻 ハンズオン（LIKEで曖昧検索）

先生が用意した事前準備コードをRunして、大量の敵データ（`enemies` テーブル）を準備してください。

```sql
/* 「ド」から始まる名前の敵を全て探す */
SELECT * FROM enemies
WHERE name LIKE 'ド%';

/* どこかに「ドラゴン」を含む敵を全て探す（前後どちらに文字があってもOK） */
SELECT * FROM enemies
WHERE name LIKE '%ドラゴン%';
```

#### 💡 解説: `%` の位置が命

- `'ド%'`: 前方一致（「ド」で始まる。ドラキー、ドラゴンなど）
- `'%ス'`: 後方一致（「ス」で終わる）
- `'%水%'`: 部分一致（どこに入っていてもOK）

> **⚠️ よくあるミス**
> `LIKE %レッド%` のように、`%` を `'`（シングルクォーテーション）の**外**に書くとエラーになります。
> `%` は必ず `'`（シングルクォーテーション）の**中**に入れてください！

### ✍️ ドリル 1-1：レッドを探せ

名前に `'レッド'` という言葉を含む敵の全データを抽出してください。

```sql
SELECT * FROM enemies
WHERE name LIKE '%レッド%';
```

> **確認：** レッドスライム、レッドドラゴン、キングレッドドラゴンの3匹が表示されましたか？
> `'%レッド%'` のように前後に `%` をつけることで「どこかに含む」検索になります！

## 🕒 レッスン 2: トップだけを取り出す `LIMIT`

100万人のゲームプレイヤーがいるとき、ランキングを見るために全員のデータを表示していたら画面がいっぱいになってしまいます。
「上位○件だけを見せて！」という便利魔法が **`LIMIT`（リミット）** です。

### 💻 ハンズオン（LIMITで件数を絞る）

先週習った `ORDER BY`（並べ替え）と組み合わせるのが最強のテクニックです。

```sql
/* HPが高い順（DESC）に並べ替えて、上位「3件だけ」を表示する */
SELECT name, hp
FROM enemies
ORDER BY hp DESC
LIMIT 3;
```

> **💡 `LIMIT` の書く位置は一番最後！**
> SQL文の順番は `SELECT` → `FROM` → `WHERE` → `ORDER BY` → `LIMIT` です。
> `LIMIT` を `SELECT` の直後に書く間違いが多いので注意！

### ✍️ ドリル 2-1：最弱モンスター決定戦

`enemies` テーブルの中で、HPが一番**低い**モンスターの「名前」と「HP」を**1匹だけ**表示してください。

```sql
SELECT name, hp
FROM enemies
ORDER BY hp ASC
LIMIT 1;
```

> **確認：** メタルスライム（HP: 5）の1匹だけが表示されましたか？
> `ASC`（小さい順）で並べて `LIMIT 1` で1件だけに絞るのがポイントです！

## 🕒 レッスン 3: カブリを消し去る `DISTINCT`

たくさんデータがあるとき、「全部で何種類の『属性（type）』があるんだろう？」と知りたいことがあります。
同じものをギュッと1つにまとめて（重複を排除して）表示する魔法が **`DISTINCT`（ディスティンクト）** です。

### 💻 ハンズオン（DISTINCTで重複排除）

```sql
/* 普通に属性を出すと、炎、炎、水、水、炎... と大量に出てしまう */
SELECT type FROM enemies;

/* DISTINCT を使うと、かぶりを消して「炎、水」など種類だけを教えてくれる！ */
SELECT DISTINCT type FROM enemies;
```

> **確認：** `DISTINCT` なしと「あり」の結果を比べてみましょう。件数が変わりましたか？

## 🚀 早く終わった人へ：ミニプロジェクト「掲示板の危険なNGワードパトロール」

あなたが管理している掲示板（`messages` テーブル）に、不適切な言葉を書き込む人が増えています！
AIパトロール隊員として、特定ワードが含まれる投稿を洗い出してください。

> **💡 `OR`（オア）とは？**
> 「AまたはB」どちらかを含む場合に使います。`AND` が「両方」なら、`OR` は「どちらか」です。

### 📝 ミッション（NGワードパトロール）

**ステップ1：** まず全メッセージを確認します。

```sql
SELECT * FROM messages;
```

**ステップ2：** 「ばか」を含む投稿だけを探します。

```sql
SELECT * FROM messages WHERE content LIKE '%ばか%';
```

**ステップ3：** `OR` を使って「ばか」または「あほ」を含む投稿を両方まとめて探します。

```sql
SELECT * FROM messages
WHERE content LIKE '%ばか%'
   OR content LIKE '%あほ%';
```

> **⚠️ 注意**
> `WHERE content LIKE '%ばか%' OR '%あほ%'` と省略するとエラーになります。
> `OR` の後ろにも必ず `content LIKE` と書く必要があります！
