# Week 12: データを操る神の手「UPDATEとDELETE」

Week 9でデータを「追加（INSERT）」し、Week 10〜11で「見る（SELECT）」ことを学びました。
今週はついに、すでにあるデータを**「書き換える（UPDATE）」**、そして完全に**「消し去る（DELETE）」**という神の操作を学びます。

---

## 🕒 レッスン 1: データを書き換える `UPDATE`

勇者がレベルアップしてHPが増えた時や、引越しで住所が変わった時に使う魔法が **`UPDATE`（アップデート）** です。

### 💻 ハンズオン（やってみよう）
先生が用意した事前準備コードをRunして、勇者テーブル（`heroes`）を作成してください。

```sql
/* UPDATE テーブル名 SET 書き換えたい列 = 新しい中身 WHERE 誰を？; */
UPDATE heroes 
SET hp = 999 
WHERE name = 'アーサー';

/* 本当に書き換わったか確認！ */
SELECT * FROM heroes;
```

> **💡 解説: `UPDATE` のルール**
> 1. `UPDATE` の後にテーブル名を指定します。
> 2. `SET` の後に `列名 = 新しいデータ` を書きます。
> 3. **絶対に `WHERE` で「誰を書き換えるか」を指定**してください！

### ✍️ ドリル 1-1：レベルアップ！
`heroes` テーブルの中にいる「ランスロット」の `level` を `50` に書き換えてください。

```sql
UPDATE heroes
SET level = ________
WHERE name = _________;
```

---

## 🕒 レッスン 2: データを消し去る `DELETE`

モンスターを倒した時や、ゲームを退会したユーザーのデータを完全に削除するのが **`DELETE`（デリート）** です。

### 💻 ハンズオン（やってみよう）
```sql
/* DELETE FROM テーブル名 WHERE 誰を？; */
DELETE FROM heroes 
WHERE name = '最弱の村人';

SELECT * FROM heroes;
```

---

## 🕒 レッスン 3: 恐怖の大事件！「WHERE忘れの全滅」を体験せよ

プロのエンジニアが一番恐れている事件、それが**「UPDATEやDELETEをする時に、間違えて WHERE を書き忘れること」**です。

### 💻 恐怖のハンズオン（最悪のバグを起こす）
もし、`WHERE`（誰を）を書き忘れるとどうなるのでしょうか…？以下のSQLを実行して、すぐに `SELECT * FROM heroes;` を見てください。

```sql
/* 【危険】WHERE を書かずに UPDATE してしまう！ */
UPDATE heroes 
SET hp = 1;

SELECT * FROM heroes;
```

**【結果の確認】**
全員のHPが `1` になってしまいました！！
これが実際の銀行のデータベースで起きたら… 全員の預金が1円になります。絶対にやってはいけない操作です。

**✨ 解決法**：事前準備コードをもう一度 Run（実行）して、壊れたデータを真っさらな初期状態にリセット（復活）させましょう！

## 🚀 今日の総仕上げ1：ミニプロジェクト「RPGステータス全回復ポイント」

ダンジョンの奥にある「回復の泉」を踏みました。
`heroes`（勇者たち）パーティのうち、レベルが10以下のやられそうなキャラのHPだけを特別に「500」に大回復してあげましょう。

### 📝 ミッション
- `UPDATE` を使って `heroes` テーブルを更新します。
- `SET` で `hp` を `500` にします。
- `WHERE` に「level が 10 以下（`<= 10`）」という条件を指定してください。
- 最後に `SELECT * FROM heroes;` で、該当するキャラのHPだけが500になっているか確認しましょう。

## 🚀 今日の総仕上げ2：ミニプロジェクト「大掃除アプリ（ゴーストユーザーの一掃）」

あなたはオンラインゲームの管理者です。サーバーが重くなってきたため、もう何年もログインしていない幽霊ユーザー（ゴーストユーザー）のデータを一斉に削除して大掃除をします。

事前準備の `users`（ユーザー）テーブルを使います。
カラムは `id`, `name`, `last_login_days`（最後にログインしてからの経過日数）です。

### 📝 ミッション
1. まず、`SELECT` で現在の全ユーザーを確認します。
2. `DELETE FROM` を使ってデータを削除します。
3. `WHERE` の条件として、「最後にログインしてからの日数（`last_login_days`）」が **365以上（1年以上放置）** になっているユーザーを指定します。
4. 最後に再び `SELECT` をして、放置ユーザーが綺麗サッパリ消えて、最近遊んでいるユーザーだけが残っていることを確認してください！
